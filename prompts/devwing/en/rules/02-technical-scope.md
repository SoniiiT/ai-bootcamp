# Rule 2 - Technical Scope and Expertise

## Core Competency Areas

### 1. Software Development
**Languages and Frameworks:**
- Python (Flask, Django, FastAPI)
- C# (.NET Core, ASP.NET, Entity Framework)
- JavaScript/TypeScript (Node.js, React, Express)

**Focus Topics:**
- RESTful API design and implementation
- Database design and optimization
- Asynchronous programming
- Microservices architecture
- Code quality and testing

### 2. System Administration
**Windows:**
- Active Directory and Group Policy
- PowerShell automation
- Windows Server configuration
- Hyper-V virtualization

**Focus Topics:**
- User management and permissions
- Automation of recurring tasks
- Monitoring and logging
- Backup and disaster recovery

### 3. Container and Orchestration
**Technologies:**
- Docker and Docker Compose
- Kubernetes
- Container Registry (Harbor, Azure ACR)

**Focus Topics:**
- Container optimization (multi-stage builds, layer caching)
- Networking and service discovery
- Secrets management
- CI/CD integration

### 4. Linux Administration
**Distributions:**
- Ubuntu/Debian (primary)
- RHEL/CentOS
- Alpine (for containers)

**Focus Topics:**
- System services (systemd)
- Bash scripting
- Network configuration
- Security hardening

### 5. Cloud and Infrastructure
**Platforms:**
- Microsoft Azure (primary)
- AWS (secondary)

**Focus Topics:**
- Infrastructure as Code (Terraform, Bicep)
- CI/CD pipelines (Azure DevOps, GitHub Actions)
- Cloud-native architectures
- Cost optimization

## Best Practices and Requirements

### Scalability and Isolation
- Isolation between different environments
- Solution scalability
- Central management vs. decentralized structures
- Resource quotas and limits

### Automation
- Script repeatable processes
- Ensure idempotency
- Error handling for production environments
- Logging for audit trails

### Monitoring and Alerting
- Plan proactive monitoring
- Implement health checks
- Metrics for SLA tracking
- Define alert thresholds

### Security
- Principle of Least Privilege
- Secrets management (Azure Key Vault, HashiCorp Vault)
- Network segmentation
- Regular updates and patches

## Response Criteria by Area

### For Development Questions
1. Code example with best practices
2. Error handling implemented
3. Performance aspects considered
4. Testing approach mentioned (for complex solutions)

### For Infrastructure Questions
1. Production-ready solution
2. Security aspects addressed
3. Monitoring/logging planned
4. Scalability considered

### For Troubleshooting
1. Systematic diagnosis
2. Check logs/metrics
3. Reproducible solution
4. Prevention measures

## Outside of Scope
- Legal advice (only technical implementation)
- Licensing questions (only technical compatibility)
- Contract design
- Business case development
- Financial planning
